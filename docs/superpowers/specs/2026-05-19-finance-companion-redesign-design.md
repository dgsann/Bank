# Дизайн: Редизайн «Банк-тамагочи» → Финансовый компаньон

Дата: 2026-05-19
Статус: утверждён пользователем (3/3 части)

## 1. Концепция и цель

Превратить игру «Банк-тамагочи» в **серьёзный финансовый компаньон**. Аватар
остаётся в центре приложения, но его состояние отражает **финансовое здоровье**
пользователя, а не игровые очки. Пользователь задаёт месячный доход, заносит
расходы чеками, категории чеков двигают «лайфстайл»-статы, жизнь в рамках
бюджета и накопления растят финансовое здоровье, активные траты в категории
открывают (заглушечные) партнёрские скидки.

### Что убирается полностью

- «Работа»/зарплата (`onSalary`)
- Кубик случайных событий 🎲 (`triggerRandomEvent`)
- Денежный бонус за уровень (level-up + `LevelUpDialog`)
- «Сон»/смена дня (`onSleep`)
- Игровые вклады и кредиты (экран «Финансы», `investMoney`/`withdrawMoney`/`takeLoan`/`repayLoan`)
- Игровой кредитный рейтинг (`creditScore`), уровни жилья (`houseLevel`), стат «энергия»
- Старый `EvolutionEngine`

### Что появляется

- Чеки: ручной ввод (рабочий поток). «Импорт с почты» и «Скан фото» —
  неактивные кнопки с меткой «скоро» (заглушки для демонстрации идеи).
- Месячный доход и контроль «вышел ли за пределы».
- Блок «Накоплено за месяц» = доход − расходы текущего месяца + цель накоплений.
  Заменяет игровые вклады/кредиты.
- Скидки по порогу трат **в категории** (по сфере, не по конкретному магазину).

## 2. Решения, принятые при брейншторме

| Вопрос | Решение |
|---|---|
| Роль аватара | Остаётся центром, отражает финансовое здоровье |
| Параметры | Главный стат «Финансовое здоровье» + лайфстайл-статы (Здоровье, Развитие, Настроение) |
| Ввод чеков | Только ручной; «почта/фото» — неактивные кнопки «скоро» |
| Контроль бюджета | Только общий месячный доход (потрачено vs доход) |
| Скидки | Только по категории трат (порог по сфере) |
| Вклады/кредиты | Убрать, заменить на «Накоплено за месяц» + цель |
| Визуальное направление | B · Dark Premium (тёмная премиальная тема) |
| Аватар-визуал | Эмодзи в кольце финансового здоровья (НЕ Canvas) |

## 3. Модель данных

`model/`

- `Receipt` — `id: Long, dateMillis: Long, store: String?, category: ReceiptCategory, amount: Double, source: ReceiptSource`
- `ReceiptCategory` — enum: `PRODUCTS` (Продукты), `CAFE` (Кафе/Рестораны),
  `TRANSPORT` (Транспорт), `HEALTH` (Здоровье/Спорт), `EDUCATION` (Образование),
  `ENTERTAINMENT` (Развлечения), `SHOPPING` (Покупки), `HOUSING` (ЖКХ/Дом),
  `OTHER` (Прочее). Каждый элемент хранит отображаемое имя и эмодзи.
- `ReceiptSource` — enum: `MANUAL`, `EMAIL`, `PHOTO` (реально используется только `MANUAL`).
- `BudgetSettings` — `monthlyIncome: Double, savingsGoal: Double`
- `AvatarStats` (вычисляемое) — `financialHealth: Int, health: Int, growth: Int, mood: Int` (все 0–100)
- `Discount` (переработан) — `id: Int, title: String, description: String,
  category: ReceiptCategory, requiredAmount: Double` + вычисляемое `unlocked`
- Удаляются: `AvatarState.kt`, `TransactionCategory.kt`

## 4. Движок `FinancialHealthEngine`

Заменяет `EvolutionEngine.kt`. Чистый объект Kotlin, без Android-зависимостей,
полностью детерминирован — пригоден для TDD.

Функции:

- `monthlySpent(receipts, year, month): Double` — сумма чеков за указанный
  календарный месяц.
- `spentByCategory(receipts, year, month): Map<ReceiptCategory, Double>`
- `financialHealth(income, monthlySpent, savingsGoal): Int` — от доли
  сбережений `(income − monthlySpent) / income`, скорректированной прогрессом к
  `savingsGoal`. Перерасход (`monthlySpent > income`) роняет показатель к нулю.
  Если доход не задан (`income <= 0`) — возвращает нейтральное `50`, а экран
  «Главная» показывает подсказку «Укажите месячный доход в Профиле».
- `lifestyleStats(receipts, year, month): AvatarStats`-часть:
  - Здоровье ← траты в `HEALTH` (+ `PRODUCTS` слегка положительно, `CAFE`
    слегка отрицательно)
  - Развитие ← траты в `EDUCATION`
  - Настроение ← траты в `ENTERTAINMENT`/`CAFE` в меру
  - Если в сфере нет трат за месяц — стат дрейфует к базовому уровню (сохраняет
    «тамагочи»-эффект, что аватар реагирует на образ жизни).
- `avatarEmoji(financialHealth): String` — ≥85 🤩, 70–84 🙂, 50–69 😐,
  30–49 😟, <30 😰.

Конкретные коэффициенты подбираются на этапе реализации под разумные значения
(шкала дохода/трат в рублях); фиксируются юнит-тестами.

## 5. Экраны и навигация

Нижняя навигация (5 пунктов): **◉ Главная · 🧾 Чеки · ＋ (центр) · 🎁 Скидки · 👤 Профиль**.

1. **Главная** — верхняя строка (месяц · ⚙), аватар-эмодзи в кольце
   «Финансовое здоровье» (значение 0–100), карточка «Потрачено / Доход» с
   прогресс-баром, карточка «Накоплено за месяц» (+ цель и %), лайфстайл-статы
   (Здоровье/Развитие/Настроение).
2. **Чеки** — заголовок, фильтр по месяцу и категории, «Итого за месяц»,
   список чеков, сгруппированный по дням (магазин · категория · сумма).
   Удаление чека: долгое нажатие → диалог подтверждения.
3. **Добавить чек** — поле суммы, выбор категории чипами, поле «Магазин
   (необязательно)», дата (по умолчанию сегодня), кнопка «Сохранить чек».
   Ниже раздел «Автоматический импорт»: неактивные кнопки «📧 Импорт с
   почты · скоро» и «📷 Скан фото чека · скоро».
4. **Скидки** — заметка «Подтягиваются из партнёрской сети банка». Карточки:
   разблокированные (выделены, кнопка «Активировать промокод» → тост-заглушка
   «Промокод XXXX скопирован (демо)»); заблокированные (прогресс к порогу по
   категории, иконка 🔒).
5. **Профиль** — поле «Месячный доход», поле «Цель накоплений в руб»,
   сводная статистика (всего чеков, потрачено с начала месяца), кнопка
   «Сбросить все данные», версия приложения.

## 6. Архитектура и файлы

`logic/FinancialHealthEngine.kt` — заменяет `EvolutionEngine.kt`.

`data/AppStorage.kt` — заменяет `AvatarStorage.kt`. SharedPreferences: список
чеков как JSON-строка (через `org.json`, встроена в Android — без новых
зависимостей) + `BudgetSettings`. Методы: `loadReceipts/saveReceipts`,
`loadBudget/saveBudget`, `clearAll`.

`presentation/MainViewModel.kt` — переписан. State: `receipts: List<Receipt>`,
`budget: BudgetSettings`, производный `AvatarStats`, список `Discount`,
`errorEvent`. Действия: `addReceipt`, `deleteReceipt`, `setMonthlyIncome`,
`setSavingsGoal`, `activateDiscount` (заглушка-тост), `resetData`.

`ui/screens/` — `HomeScreen.kt`, `ReceiptsScreen.kt`, `AddReceiptScreen.kt`,
`DiscountsScreen.kt`, `ProfileScreen.kt`; общий компонент `AvatarRing`
(эмодзи в кольце). `MainActivity.kt` — только `Scaffold` + нижняя навигация +
роутинг между экранами.

`ui/theme/Color.kt` + `Theme.kt` — тёмная премиальная палитра: фон `#0E1116`,
карточки `#171C23`, границы `#242B34`, акцент `#34E29B`, доп. `#FF7A59`
(Здоровье) и `#5B8DEF` (Развитие), текст `#E8ECF1`. `darkColorScheme`,
`dynamicColor = false` (фиксированный фирменный вид).

### Поток данных

UI → ViewModel (действие) → обновляет `receipts`/`budget` → `AppStorage`
сохраняет → `FinancialHealthEngine` пересчитывает `AvatarStats` и состояние
скидок из чеков текущего месяца → `StateFlow` → UI.

### Обработка ошибок

Валидация только на границе ввода: сумма чека > 0, доход ≥ 0, цель ≥ 0 →
`errorEvent` → Toast. Внутренние пересчёты движка без защитного кода
(детерминированы).

## 7. Тестирование

- **TDD** для `FinancialHealthEngine` (JUnit, чистый Kotlin):
  - экономия → рост финансового здоровья; перерасход → падение
  - `income == 0` не роняет приложение
  - траты в категории → соответствующий лайфстайл-стат
  - `monthlySpent` учитывает только текущий календарный месяц
  - порог трат по категории → разблокировка скидки
- UI проверяется вручную в Android Studio. **В среде разработки эмулятор
  недоступен — автоматическая проверка UI невозможна, это будет указано явно
  при сдаче.**

## 8. Миграция данных

Старые ключи SharedPreferences (`fintama_prefs`) несовместимы с новой моделью.
При первом запуске новой версии старого прогресса нет — чистый старт. Для
игрового демо допустимо. Опционально: при инициализации `AppStorage` очистить
устаревшие ключи.

## 9. Вне области (YAGNI)

- Реальная интеграция с почтой/камерой/OCR (только неактивные заглушки)
- Лимиты по отдельным категориям и авторазбивка бюджета (только общий доход)
- Скидки по конкретному магазину (только по категории)
- Реальные финансовые продукты (вклады/кредиты убраны)
- Многомесячная аналитика/графики истории
